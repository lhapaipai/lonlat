# RTerrain

## Usage

```tsx
import { RMap, RSource, RTerrain } from "maplibre-react-components";

const rasterDemTiles = ["https://s3.amazonaws.com/elevation-tiles-prod/terrarium/{z}/{x}/{y}.png"];

export default function App() {
  return (
    <RMap>
      <RSource
        type="raster-dem"
        id="terrarium"
        tiles={rasterDemTiles}
        encoding="terrarium"
        tileSize={256}
      />
      <RTerrain source="terrarium" exaggeration={1.3} />
    </RMap>
  );
};

```

## Reference

`RTerrain` accept same props as `TerrainSpecification`, all his props are reactives.

```ts
type RTerrainProps = TerrainSpecification;

type TerrainSpecification = {
	"source": string;
	"exaggeration"?: number;
};
```


## Incompatibility `RTerrain` / `RTerrainControl`.

The `RTerrainControl` is only a wrapper of MapLibre `TerrainControl`.
When you use this control, the maplibre instance will be modified without
your RTerrain instance being aware of it, which can lead to incompatibilities.
So use either `RTerrain` or `RTerrainControl` but never both components at the
same time.

Generally speaking, you use `RTerrain` if you want to benefit from reactivity
 with the exaggeration option.
